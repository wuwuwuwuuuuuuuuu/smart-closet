# 智能提醒后端切换智谱知识库多模态检索实施方案

## 1. 问题复述

当前项目的“智能推荐 / 智能提醒”页面已经基本完成前端页面改版，但后端推荐链路仍然停留在旧的“直接调用多模态模型 + 本地回退”的阶段，没有形成稳定的“用户衣橱 → 检索 → 推荐 → 试穿落地”闭环。

你现在要做的是：

1. 将现有智能推荐后端从“直接分析整组衣橱图片”的实现，升级为**基于智谱 BigModel 知识库的多模态检索方案**；
2. 优先采用你提出的**方案二：多知识库隔离**，即：
   - 用户 A → 知识库 A
   - 用户 B → 知识库 B
   - 用户 C → 知识库 C
3. 保留当前项目已有页面、数据库、试穿页逻辑，不重建系统；
4. 让智能推荐结果最终仍然能够回到当前项目已经存在的 `selectedClothesIds` 链路，确保试穿页可以继续自动选衣和摆放。

本次不是直接改代码，而是先给出一份**可以交给另一个 agent 直接实施的、足够详细的真实改造方案**。

## 3. 官方文档约束与可落地结论

以下结论基于智谱官方文档整理：

1. 智谱提供**知识库管理 API**，支持：
   - 创建知识库
   - 查询知识库列表
   - 上传文件文档
   - 全模态知识库检索
2. 创建知识库接口为：
   - `POST /api/llm-application/open/knowledge`
3. 知识库列表接口为：
   - `GET /api/llm-application/open/knowledge`
4. 上传文件文档接口为：
   - `POST /api/llm-application/open/document/upload_document/{id}`
5. 全模态知识库检索接口为：
   - `POST /api/zrag/retrieval/retrieve`
6. 全模态检索支持：
   - 文本 query
   - `multimodal_parts`
   - 多知识库 `knows`
   - `top_k / top_n`
   - `recall_method`
7. 返回结果里会包含：
   - `doc_id`
   - `image_url`
   - `medias`
   - `score`
   - `rerank_score`
   - `metadata`
8. 智谱官方速率限制文档明确提到：
   - 不同模型存在独立并发限制
   - 可能出现用户限流错误 `1302`
   - 可能出现平台过载错误 `1305`
   - 建议使用请求队列、重试间隔和降级机制

### 3.1 基于文档的工程结论

结合当前项目，最适合的接入方式不是“知识库直接替代数据库”，而是：

```text
本地 clothes 数据库 = 业务主数据
智谱 knowledge base = 检索增强层
```

原因：

1. 当前试穿页依赖本地 `clothes._id`
2. 当前衣物图片、分类、季节、标签都已经在本地 `clothes` 中
3. 智谱检索结果只有 `doc_id / metadata / media`，不能天然替代试穿页所需的本地衣物实体
4. 因此必须保留：
   - 本地衣物主表
   - 本地与知识库文档之间的映射关系

所以本次改造的正确方向是：

```text
新增“用户知识库绑定 + 衣物知识库同步 + 检索结果映射层”
而不是替换现有数据库架构
```

## 4. 总体改造方案

## 4.1 新的后端链路

建议改造成下面这条链路：

```text
pages/daily/daily.js
  → wx.cloud.callFunction('smartRecommendPhoto')
    → 读取当前用户 openid / user_id
    → 获取或校验该用户绑定的 knowledge_id
    → 查询本地 clothes 集合
    → 检查衣物是否已同步到知识库
    → 用用户 query + 天气 + 场景 发起知识库检索
    → 拿到检索结果 contents[]
    → 将 doc_id / metadata 映射回本地 clothes._id
    → 组装 recommendationResult
    → 返回 selectedClothesIds 给前端
    → 前端继续复用现有 tryon 自动摆放能力
```

```js
const { logWarning } = require('./logger')

function uniqueStringList(list = []) {
  if (!Array.isArray(list)) {
    logWarning('recommendationComposer.uniqueStringList', 'invalid list', {
      listType: typeof list
    })
    return []
  }

  return [...new Set(
    list
      .filter(Boolean)
      .map(item => String(item).trim())
      .filter(Boolean)
  )]
}
```

## 子任务 2：建立“用户 ↔ 知识库”绑定层

### 1. 任务目标

让每个用户都能绑定自己的 `knowledge_id`，实现多知识库隔离。

### 2. 实现方案

建议在 `smartRecommendPhoto` 下新增绑定模块：

```js
// cloudfunctions/smartRecommendPhoto/utils/knowledge-binding.js
const { logError, logWarning } = require('./logger')

function normalizeKnowledgeStatus(status) {
  const allowed = ['unbound', 'creating', 'ready', 'failed']
  return allowed.includes(status) ? status : 'unbound'
}

async function ensureUserKnowledgeBinding({ db, openid, provider }) {
  const userRes = await db.collection('users')
    .where({ _openid: openid })
    .orderBy('createdAt', 'desc')
    .get()

  if (!userRes.data.length) {
    logWarning('knowledgeBinding.ensureUserKnowledgeBinding', 'user not found', { openid })
    throw new Error('user not found')
  }

  const user = userRes.data[0]
  const knowledgeId = typeof user.knowledge_id === 'string' ? user.knowledge_id.trim() : ''
  const knowledgeStatus = normalizeKnowledgeStatus(user.knowledge_status)

  if (knowledgeId && knowledgeStatus === 'ready') {
    return {
      user,
      knowledgeId,
      created: false
    }
  }

  const created = await provider.createKnowledgeBase({
    name: `wardrobe_${user._id}`,
    description: `smart-closet user ${user._id}`
  })

  if (!created || !created.id) {
    logError('knowledgeBinding.ensureUserKnowledgeBinding', new Error('knowledge create failed'))
    throw new Error('knowledge create failed')
  }

  await db.collection('users').doc(user._id).update({
    data: {
      knowledge_id: created.id,
      knowledge_status: 'ready',
      knowledge_bound_at: db.serverDate(),
      knowledge_provider: 'zhipu'
    }
  })

  return {
    user,
    knowledgeId: created.id,
    created: true
  }
}
```

`smartRecommendPhoto/index.js` 开头改成先拿用户，再拿知识库绑定。

### 3. 计算正确性、异常处理与单元测试

这里的关键计算是状态归一化与“是否需要创建知识库”的判断。

要求：

- `knowledge_status` 不是允许值时，降级到 `unbound`
- 已存在 `knowledge_id + ready` 时，不能重复创建知识库
- 创建失败时必须 `logError`

建议测试：

```js

assert.strictEqual(normalizeKnowledgeStatus('ready'), 'ready')
assert.strictEqual(normalizeKnowledgeStatus('abc'), 'unbound')
```

### 4. 验证此任务成功完成的方法

验证标准：

- 新用户首次调用智能推荐时，`users` 中能自动写入 `knowledge_id`
- 老用户再次请求时不会重复创建
- 用户 A 与用户 B 的 `knowledge_id` 不相同

### 5. 迭代开发要求

必须基于现有 `users` 集合扩展字段，不能额外强制引入新的用户主表或重做登录系统。

## 子任务 4：实现智谱知识库 API 适配层

### 1. 任务目标

把智谱知识库相关 API 封装成统一 provider，避免把 HTTP 细节散落在业务逻辑中。

### 2. 实现方案

建议新增：

```js
// cloudfunctions/smartRecommendPhoto/utils/zhipu-knowledge-provider.js
const axios = require('axios')
const { logError } = require('./logger')

function getProviderConfig() {
  return {
    apiKey: process.env.ZHIPU_API_KEY || '',
    baseUrl: (process.env.ZHIPU_BASE_URL || 'https://open.bigmodel.cn/api').replace(/\/$/, '')
  }
}

async function request(method, path, data, config = {}) {
  const { apiKey, baseUrl } = getProviderConfig()
  if (!apiKey) {
    throw new Error('missing ZHIPU_API_KEY')
  }

  try {
    const res = await axios({
      method,
      url: `${baseUrl}${path}`,
      data,
      headers: {
        Authorization: `Bearer ${apiKey}`,
        ...config.headers
      },
      timeout: config.timeout || 30000
    })
    return res.data
  } catch (error) {
    logError('zhipuProvider.request', error, {
      path,
      status: error && error.response && error.response.status
    })
    throw error
  }
}

async function createKnowledgeBase(payload) {
  return request('post', '/llm-application/open/knowledge', {
    embedding_id: 11,
    name: payload.name,
    embedding_model: 'Embedding-3',
    contextual: 1,
    description: payload.description || ''
  })
}

async function retrieveFromKnowledge(payload) {
  return request('post', '/zrag/retrieval/retrieve', {
    multimodal: true,
    knows: [{ id: payload.knowledgeId }],
    query: payload.query,
    multimodal_parts: payload.multimodalParts || [],
```

## 子任务 5：把 `smartRecommendPhoto` 改造成“知识库检索优先”的推荐主流程

### 1. 任务目标

让当前主云函数从“直接把整组图片丢给多模态模型”改为“知识库检索优先 + 本地兜底推荐”的稳定架构。

### 2. 实现方案

建议重构 `cloudfunctions/smartRecommendPhoto/index.js` 主流程：

```js
exports.main = async (event) => {
  try {
    const wxContext = cloud.getWXContext()
    const openid = wxContext.OPENID
    const userQuery = normalizeInput(event.userQuery)

    if (!userQuery) {
      logWarning('smartRecommendPhoto.main', 'missing userQuery')
      return { code: 400, message: 'userQuery 不能为空' }
    }

    const provider = getKnowledgeProvider('zhipu')
    const binding = await ensureUserKnowledgeBinding({ db, openid, provider })
    const clothesRes = await db.collection('clothes').where({ user_id: binding.user._id }).get()
    const clothesList = Array.isArray(clothesRes.data) ? clothesRes.data : []

    const readyClothes = clothesList.filter(item => item.knowledge_sync_status === 'ready' && item.knowledge_doc_id)
    if (!readyClothes.length) {
      logWarning('smartRecommendPhoto.main', 'no knowledge-ready clothes', { userId: binding.user._id })
      return buildFallbackResponse(event, clothesList)
    }

    const retrievalRes = await provider.retrieveFromKnowledge({
      knowledgeId: binding.knowledgeId,
      query: buildRecommendationQuery(event),
      topK: 8,
      topN: 4
    })

    const retrievalItems = parseKnowledgeRetrievalResult(retrievalRes)
    const selectedClothesIds = mapRetrievalToClothesIds(retrievalItems, clothesList)

    if (!selectedClothesIds.length) {
      logWarning('smartRecommendPhoto.main', 'knowledge result mapped empty', {
        knowledgeId: binding.knowledgeId
      })
      return buildFallbackResponse(event, clothesList)
    }

    return {
      code: 200,
      message: 'ok',
      data: composeRecommendationResult({
        requestId: event.requestId,
        summary: buildSummaryFromRetrieval(retrievalItems, event),
        replyText: buildReplyFromRetrieval(retrievalItems, event),
        outfitLines: buildOutfitLines(retrievalItems),
        tips: buildTips(event),
        selectedClothesIds,
        knowledgeId: binding.knowledgeId,
        retrievalHitCount: retrievalItems.length,
        retrievalSource: 'zhipu_knowledge'
      })
    }
  } catch (error) {
    logError('smartRecommendPhoto.main', error)
    return {
      code: 500,
      message: '智能推荐生成失败',
      error: error.message
    }
  }
}
```

推荐策略建议：

```text
优先知识库检索
→ 检索结果可映射则返回真实衣物
→ 映射失败则降级为 metadata/mock 推荐
→ 但返回结构保持一致
```
