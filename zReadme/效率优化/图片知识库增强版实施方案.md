# 图片知识库增强版实施记录

## 目标

把智能推荐从“文字知识文档优先”升级为“图片向量知识库优先”，让用户衣橱图片直接参与检索、理解和推荐；保留现有文字知识库/前端 mock 作为降级路径。

## 已实现内容

1. 衣物图片字段补齐
   - `cloudfunctions/addClothing/index.js` 保存 `originalImage`。
   - 新增 `image_knowledge_status`、`image_embedding_status`、`image_embedding_error`、`image_embedding_updated_at`。
   - `cloudfunctions/updateClothing/index.js` 在图片变化时重置图片知识库状态。
   - 公共纯函数：`cloudfunctions/common/clothing-image-fields.js`。

2. 图片向量工具层
   - `cloudfunctions/common/image-vector-utils.js`：向量合法性、余弦相似度、TopK 排序。
   - `cloudfunctions/common/dashscope-multimodal-provider.js`：DashScope 多模态 embedding 与兼容模式 chat 调用。

3. 图片知识库重建
   - `cloudfunctions/rebuildUserKnowledgeBase/index.js`：读取衣橱图片、获取临时 URL、生成图片 embedding、写入 `clothes_image_vectors`、回写衣物状态。
   - `cloudfunctions/rebuildUserKnowledgeBase/rebuild.helpers.js`：limit 与统计摘要纯函数。

4. 图片检索优先推荐
   - `cloudfunctions/smartRecommendPhoto/index.js`：文本 embedding → 图片向量 TopK → 多模态推荐 → 返回前端兼容结构。
   - 降级路径：无 API Key、无向量、无命中、模型 JSON 异常时返回 `retrievalSource: fallback` 或模型降级结果。

5. 推荐到试穿页闭环
   - `pages/tryon/tryon.js` 恢复使用 `tryon.helpers.js`。
   - 根据 `selectedClothesIds` 匹配真实衣物并自动摆放到画板。

6. 前端状态展示
   - `pages/daily/daily.helpers.js` 保留 `retrievalSource`、`retrievalHitCount`、`fallbackReason`。
   - `pages/daily/daily.wxml/.wxss` 在推荐卡片显示来源和命中数量。

## 真机验证路径

1. 上传一件衣物，确认 `clothes` 中出现图片知识库状态字段。
2. 智能推荐页点击“补同步/查状态”。
3. 确认 `clothes_image_vectors` 出现向量记录。
4. 输入穿搭需求，推荐卡片应展示 `来源：image_vector · 命中 N 件`。
5. 点击推荐卡片进入试穿页，推荐衣物应自动出现在画板。

## 注意事项

- 云函数需要配置有效 `DASHSCOPE_API_KEY`；否则会自动降级。
- 当前检索采用云函数内 JS 余弦相似度，适合小规模私人衣橱。
- 现有 Markdown 百炼知识库能力未删除，后续可继续作为 fallback。
