# 智能提醒模块流程优化方案

## 1. 问题复述

当前项目需要把“智能提醒”从**静态天气文案**升级成一条真正可执行的 AI 流程：

> 用户输入需求 → 大模型理解需求 → 分析衣橱照片 → 识别衣物特征 → 智能搭配推荐 → 生成搭配照片

你已经明确希望使用**百度千帆 / 文心一言多模态能力**实现该流程，并给出了一个目标式伪代码：先分析衣橱照片，再结合用户需求做推荐，最后生成搭配照片。

本次工作的重点不是直接写最终代码，而是：

1. **结合当前项目真实代码结构**，给出可落地的修改方案；
2. **把任务拆成一批足够小、一次能稳妥完成的子任务**；
3. 每个子任务都要说明：
   - 目标
   - 具体改法
   - 校验 / 错误日志 / 单测
   - 成功验证方式
   - 必须基于现有项目迭代，而不是另起炉灶

## 2. 当前项目实际情况（基于仓库现状）

### 2.1 当前前端结构

当前 `app.json` 里**没有启用 subpackages**，所有页面仍是主包页面，因此这次不要做分包级重构，而应该继续沿用当前结构，做**增量迭代**。

当前与“智能提醒 / 穿搭推荐”最相关的页面：

- `pages/daily/daily.js`：已有天气获取逻辑，天然适合承接“智能提醒”
- `pages/wardrobe/wardrobe.js`：可读取用户衣橱
- `pages/uploadClothes/uploadClothes.js` + `pages/clothingInfo/clothingInfo.js`：完成衣物上传和基础信息录入
- `pages/preview/preview.js`：可用于查看最终生成图，但需要修一下参数兼容
- `pages/home/home.js`：已有“每日穿搭”入口

### 2.2 当前后端 / 云函数结构

项目后端是**微信云开发 + 云函数**模式，现有模式已经比较明确：

- `cloudfunctions/login`
- `cloudfunctions/addClothing`
- `cloudfunctions/getClothesList`
- `cloudfunctions/aiTryon`

其中 `cloudfunctions/aiTryon/index.js` 已经是一个非常重要的参考样板：

1. 前端使用 `wx.cloud.callFunction`
2. 云函数内使用 `axios` 调第三方 AI 接口
3. 需要把 `cloud://` 文件先转成临时 HTTPS URL

所以智能提醒最适合新增为**独立云函数**，例如：

- `cloudfunctions/smartReminder`

### 2.3 当前代码中的真实约束与问题

#### 约束 A：衣橱图片在数据库里存的是 `cloud://` fileID
这意味着：
- 发给文心一言之前，必须先通过 `cloud.getTempFileURL` 转成临时 HTTPS 地址；
- 这一步和 `aiTryon` 的处理逻辑一致，可以直接复用思路。

#### 约束 B：`daily` 页目前只是“天气页”，还不是“AI 智能提醒页”
`pages/daily/daily.js` 已经拿到了：
- `weatherInfo`
- `currentCity`
- `currentOotdContent`

但是 `pages/daily/daily.wxml` 里大量内容还是**硬编码**，并没有真正绑定 `weatherInfo`、也没有用户需求输入框，更没有推荐结果卡片和生成图区域。

#### 约束 C：当前衣物信息字段不够支撑精细推荐
`clothes` 集合目前主要字段是：

```js
{
  _openid,
  user_id,
  name,
  image,
  season,
  category,
  tags,
  material,
  brand,
  created_at,
  updated_at
}
```

缺少：
- 颜色
- 风格
- 版型
- 场景
- 搭配禁忌
- AI 自动识别结果缓存

所以如果每次都重新分析全部衣橱，会慢、贵、且不稳定。**必须加 AI 分析缓存层。**

#### 约束 D：当前项目没有公共工具层，也没有 `logError / logWarning`
仓库里目前没有统一日志工具，也没有测试基础设施。
因此建议在新云函数内部新增：

- `utils/logger.js`
- `utils/validate.js`
- `tests/*.test.js`

注意：**只给新模块补齐，不要对整个老项目做大清洗。**

#### 约束 E：`preview` 现有参数协议有一个真实 bug
- `pages/tryon/tryon.js` 跳转时传的是 `img`
- `pages/preview/preview.js` 只识别 `aiImage` / `productImage`

如果后续智能提醒生成图也想跳到 `preview` 页，需要顺手把这个兼容问题补掉，否则容易出现“生成成功但预览为空”。

## 3. 本次推荐的实际代码改造方案

## 3.1 总体落地原则

本次不要新建一整套新架构，而是**贴着现有微信小程序 + 云函数架构做迭代**：

1. **页面层**：继续用 `pages/daily` 作为智能提醒承载页；

# 子任务 1：建立智能提醒云函数入口、参数校验与统一日志

## 1. 任务目标

新增 `cloudfunctions/smartReminder` 云函数，建立稳定入口，先把“请求校验、用户识别、统一响应、错误日志”打牢，为后续 AI 接入提供干净边界。

## 2. 实现方案

### 2.1 新增文件

- `cloudfunctions/smartReminder/index.js`
- `cloudfunctions/smartReminder/package.json`
- `cloudfunctions/smartReminder/utils/logger.js`
- `cloudfunctions/smartReminder/utils/validate.js`
- `cloudfunctions/smartReminder/utils/constants.js`

### 2.2 入口云函数建议写法

```js
const cloud = require('wx-server-sdk')
const { validateSmartReminderRequest } = require('./utils/validate')
const { logError, logWarning } = require('./utils/logger')
const { runSmartReminder } = require('./services/recommendation-service')

cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })

exports.main = async (event) => {
  const wxContext = cloud.getWXContext()
  const openid = wxContext.OPENID

  try {
    const payload = validateSmartReminderRequest(event)

    if (!payload.isValid) {
      logWarning('validateSmartReminderRequest', 'invalid request', {
        reasons: payload.errors
      })
      return {
        code: 400,
        message: '智能提醒请求参数不完整',
        errors: payload.errors
      }
    }

    const result = await runSmartReminder({
      openid,
      payload: payload.value,
      cloud
    })

    return {
      code: 200,
      message: '智能提醒成功',
      data: result
    }
  } catch (error) {
    logError('smartReminder.main', error)
    return {
      code: 500,
      message: '智能提醒失败',
      error: error.message
    }
  }
}
```

### 2.3 日志工具必须简练

```js
function logError(scope, error, extra = {}) {
  console.error(`[SmartReminder][${scope}]`, {
    message: error && error.message ? error.message : String(error),
    ...extra
  })
}

function logWarning(scope, message, extra = {}) {
  console.warn(`[SmartReminder][${scope}] ${message}`, extra)

```

- 所有成功路径不打印冗余日志，只在**校验失败 / 异常 / 模型返回不可解析**时记录。

# 子任务 2：实现衣橱照片聚合、临时 URL 转换与 AI 特征缓存

## 1. 任务目标

从当前用户的 `clothes` 集合中读取衣橱，筛出可用图片，把 `cloud://` 转成临时 HTTPS URL，并建立 `aiAnalysis` 缓存，避免每次请求都全量跑多模态分析。

## 2. 实现方案

### 2.1 新增文件

- `cloudfunctions/smartReminder/services/wardrobe-service.js`

### 2.2 读取衣橱时沿用现有字段

```js
async function loadWardrobeByOpenid({ db, openid, cloud }) {
  const userRes = await db.collection('users').where({ _openid: openid }).get()
  if (!userRes.data.length) {
    throw new Error('用户不存在，无法读取衣橱')
  }

  const userId = userRes.data[0]._id
  const clothesRes = await db.collection('clothes')
    .where({ user_id: userId })
    .orderBy('created_at', 'desc')
    .get()

  const rawItems = clothesRes.data.filter(item => !!item.image)
  const fileList = rawItems.map(item => item.image)
  const tempRes = await cloud.getTempFileURL({ fileList })

  return rawItems.map((item, index) => ({
    ...item,
    tempImageUrl: tempRes.fileList[index]?.tempFileURL || ''
  }))
}
```

### 2.3 只分析“未命中缓存 / 缓存过期”的衣物

```js
function splitCachedAndPending(items, now = Date.now()) {
  const pending = []
  const cached = []

  items.forEach(item => {
    const updatedAt = item.aiAnalysis?.updatedAt
      ? new Date(item.aiAnalysis.updatedAt).getTime()
      : 0

    const isFresh = item.aiAnalysis?.status === 'ready' && (now - updatedAt) < 30 * 24 * 3600 * 1000

    if (isFresh) cached.push(item)
    else pending.push(item)
  })

  return { cached, pending }
}
```

### 2.4 批量控制

不要把整个衣橱一次性全部丢给多模态模型。建议：

- 默认每批 `4 ~ 6` 张图；
- 先缓存命中复用；
- 对 pending 项分批分析；
- 分析完成后，把结构化结果写回原始 `clothes` 文档的 `aiAnalysis` 字段。

### 2.5 写回缓存建议

await db.collection('clothes').doc(itemId).update({

## 3. 计算正确性、异常处理、日志与单测

### 3.1 必须判断的计算是否正确

- `rawItems.length` 是否 > 0

- 批次数量是否正确（如 13 件待分析 → 3 批：6、6、1）

# 子任务 3：封装百度千帆 / 文心一言多模态适配器

## 1. 任务目标

把“衣物特征识别”和“结合用户需求做搭配推荐”封装成统一的文心一言服务层，避免 AI 调用逻辑散落在 `index.js`。

### 2.1 新增文件

- `cloudfunctions/smartReminder/services/qianfan-client.js`
- `cloudfunctions/smartReminder/services/ernie-multimodal.js`
- `cloudfunctions/smartReminder/utils/json.js`

### 2.2 鉴权策略

**不要把 API Key 硬编码进仓库。**
建议使用云函数环境变量：

- `QIANFAN_API_KEY`

### 2.3 通用请求客户端

```js
const axios = require('axios')

async function qianfanPost({ path, data }) {
  const apiKey = process.env.QIANFAN_API_KEY
  if (!apiKey) throw new Error('缺少 QIANFAN_API_KEY')

  const res = await axios.post(`https://qianfan.baidubce.com${path}`, data, {
    headers: {
      Authorization: `Bearer ${apiKey}`,
      'Content-Type': 'application/json'
    },
    timeout: 30000
  })

  return res.data
}
```

### 2.4 分析衣橱照片：输出严格 JSON

async function analyzeWardrobePhotos({ photos }) {

### 2.5 搭配推荐：输入结构化衣橱特征，而不是原始杂乱文案

```js
async function recommendFromPhotos({ userQuery, analyzedItems, userPreferences, weather, occasion }) {

    userQuery,

        }]
      }],
      temperature: 0.3,
      response_format: { type: 'json_object' }
    }
  })

  return safeParseModelJson(data)
}
```

### 2.6 必须做“模型输出兜底解析”

模型偶尔会：
- 包 ```json code fence
- 返回字段不全
- 返回选择了不存在的衣物 ID

所以 `safeParseModelJson` 必须负责：
- 提取 JSON 主体
- 失败时抛异常
- 对缺字段做默认值补齐

### 2.4 图片生成服务建议代码框架

```js
async function generateOutfitFromPhotos({ recommendation, weatherInfo, userQuery }) {
  if (!recommendation?.selectedItems?.length) {
    return { status: 'skipped', url: '' }
  }

  const prompt = buildOutfitPrompt({ recommendation, weatherInfo, userQuery })

  const createRes = await qianfanPost({
    path: '/beta/image/qianfan-image-v1',
    data: {
      model: process.env.QIANFAN_IMAGE_MODEL,
      prompt,
      image_list: recommendation.selectedItems.map(item => item.image)
    }
  })

  const taskId = createRes.data?.task_id
  return await pollImageTask(taskId)
}
```

### 2.5 轮询任务状态

```js
async function pollImageTask(taskId) {
  const maxRetry = 20
  for (let i = 0; i < maxRetry; i++) {
    await sleep(2000)
    const taskRes = await qianfanPost({
      path: '/beta/image/query',
      data: { task_id: taskId }
    })

    const status = taskRes.data?.task_status
    if (status === 'SUCCESS') {
      return {
        status: 'success',
        url: taskRes.data?.sub_task_result_list?.[0]?.final_image_list?.[0]?.img_url || ''
      }
    }
    if (status === 'FAIL') {
      throw new Error(taskRes.data?.error_msg || '搭配图生成失败')
    }
  }

  return { status: 'timeout', url: '' }
}
```

> 注：这里的图片接口字段名，最终实现时要以当前控制台可用模型对应的官方 API 文档为准；如果该内置模型并非多参考图模型，就走上面的方案 B。

async requestSmartReminder() {
  const { userQuery, selectedOccasion, weatherInfo } = this.data

  if (!userQuery.trim()) {
    wx.showToast({ title: '请先输入你的穿搭需求', icon: 'none' })
    return
  }

  this.setData({
    smartReminderLoading: true,
    smartReminderError: '',
    smartReminderResult: null
  })

  try {
    const res = await wx.cloud.callFunction({
      name: 'smartReminder',
      data: {
        userQuery,
        occasion: selectedOccasion,
        weatherInfo,
        userPreferences: {}
      }
    })

    if (res.result.code !== 200) {
      throw new Error(res.result.message || '智能提醒失败')
    }

    this.setData({
      smartReminderLoading: false,
      smartReminderResult: res.result.data
    })
  } catch (error) {
    this.setData({
      smartReminderLoading: false,
      smartReminderError: error.message
    })
    wx.showToast({ title: '智能提醒失败，请稍后重试', icon: 'none' })

}
