# 智能提醒模块流程优化实施方案（多模态衣橱推荐链路）

## 1. 问题复述

当前项目已经完成了“智能推荐”页面和“试穿”页面的基础 UI，但“智能提醒 / 智能推荐”的核心业务链路还没有真正打通。

目标流程不是简单地在对话框里返回一段文字，而是要实现下面这条完整链路：

```text
用户输入需求
→ 大模型理解需求
→ 分析衣橱照片
→ 识别衣物特征
→ 生成智能搭配推荐
→ 从衣橱中选取满足需求的衣物
→ 跳转试穿页并自动摆放这些衣物
```

本次不把方案绑定到单一模型厂商，而是采用**“多模态模型适配层”**思路：让模型基于衣橱图片做图像理解，再结合用户需求、天气、场合、偏好生成推荐结果，并把选中的衣物 `_id` 回传给前端，复用现有试穿页自动选衣能力。

第一版可以先按百度千帆 / 文心一言落地，但后续如果改成其他模型，只替换服务层实现，前端和主云函数尽量不动。

你要求的不是立刻改代码，而是：

1. 基于当前真实项目状态，给出可落地的代码修改方案；
2. 将任务拆解成若干个一次性可稳定交付的子任务；
3. 每个子任务都必须包含：
   - 任务目标
   - 详细实现方案（含计划写的代码块）
   - 计算正确性 / 错误日志 / 单元测试要求
   - 验证完成的方法
   - 必须基于当前架构迭代，而不是另起炉灶

# 子任务 3：新增云函数 `smartRecommendPhoto`，在服务端完成用户衣橱查询、临时 URL 转换和模型调用入口

## 1. 任务目标

把“智能推荐”真正接到后端，形成统一服务入口，前端只负责发请求和渲染结果。

## 2. 实现方案

### 2.1 涉及文件

- `cloudfunctions/smartRecommendPhoto/index.js`（新增）
- `cloudfunctions/smartRecommendPhoto/package.json`（新增）
- `cloudfunctions/config.json`（补充触发器，可选）
- `pages/daily/daily.js`

### 2.2 云函数职责

云函数必须承担以下职责：

1. 校验用户身份；
2. 根据 `_openid` / `user_id` 读取当前用户衣橱；
3. 把 `cloud://` 文件 ID 转成模型可访问的临时 HTTPS URL；
4. 调用 `utils/model-provider.js`；
5. 把模型结果映射回数据库衣物 `_id`；
6. 返回结构化推荐结果；
7. 失败时返回稳定错误结构。

### 2.3 计划代码框架

```js
const cloud = require('wx-server-sdk')
const { logError, logWarning } = require('./utils/logger')
const { buildWardrobePhotoPayload, mapPhotoUrlsToClothesIds } = require('./utils/wardrobe-photo-mapper')
const ernieMultimodal = require('./utils/ernie-multimodal')

cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })
const db = cloud.database()

exports.main = async (event) => {
  try {
    const wxContext = cloud.getWXContext()
    const openid = wxContext.OPENID
    const { userQuery, weatherInfo, weatherSuggestion, city, occasion, userPreferences } = event

    if (!userQuery || !String(userQuery).trim()) {
      return { code: 400, message: 'userQuery 不能为空' }
    }

    const userRes = await db.collection('users').where({ _openid: openid }).orderBy('createdAt', 'desc').get()
    if (!userRes.data.length) {
      return { code: 404, message: '用户不存在' }
    }

    const userId = userRes.data[0]._id
    const clothesRes = await db.collection('clothes').where({ user_id: userId }).orderBy('created_at', 'desc').get()
    const wardrobeItems = buildWardrobePhotoPayload(clothesRes.data)

    if (!wardrobeItems.length) {
      return { code: 422, message: '衣橱为空，无法生成推荐' }
    }

    const tempUrlRes = await cloud.getTempFileURL({
      fileList: wardrobeItems.map(item => item.photoFileId)
    })

    const wardrobePhotos = tempUrlRes.fileList.map((file, index) => ({
      clothesId: wardrobeItems[index].clothesId,
      photoUrl: file.tempFileURL,
      name: wardrobeItems[index].name,
      category: wardrobeItems[index].category,
      season: wardrobeItems[index].season,
      tags: wardrobeItems[index].tags
    }))

    const modelResult = await modelProvider.photoBasedRecommendation({
      userQuery,
      wardrobePhotos,
      userPreferences,
      weatherInfo,
      weatherSuggestion,
      city,
      occasion
    })

    return {
      code: 200,
      data: modelResult
    }
  } catch (error) {
    logError('smartRecommendPhoto.main', error)
    return {
      code: 500,
      message: '智能推荐生成失败',
      error: error.message
    }
  }
}
```

```js
function parseStructuredRecommendation(modelText, wardrobePhotos = []) {
  if (!modelText || typeof modelText !== 'string') {
    throw new Error('modelText 不能为空')
  }

  const jsonMatch = modelText.match(/\{[\s\S]*\}/)
  if (!jsonMatch) {
    throw new Error('模型输出中未找到 JSON')
  }

  const parsed = JSON.parse(jsonMatch[0])
  const validPhotoUrls = new Set(wardrobePhotos.map(item => item.photoUrl))
  const selectedPhotoUrls = Array.isArray(parsed.selectedPhotoUrls)
    ? parsed.selectedPhotoUrls.filter(url => validPhotoUrls.has(url))
    : []

  return {
    summary: parsed.summary || '',
    replyText: parsed.replyText || '',
    outfitLines: Array.isArray(parsed.outfitLines) ? parsed.outfitLines.filter(Boolean) : [],
    tips: Array.isArray(parsed.tips) ? parsed.tips.filter(Boolean) : [],
    selectedPhotoUrls
  }
}
```

```js
function mapPhotoUrlsToClothesIds(selectedPhotoUrls = [], wardrobePhotos = []) {
  const urlToIdMap = new Map(
    wardrobePhotos
      .filter(item => item.photoUrl && item.clothesId)
      .map(item => [item.photoUrl, String(item.clothesId)])
  )

  return [...new Set(selectedPhotoUrls)]
    .map(url => urlToIdMap.get(url))
    .filter(Boolean)
}
```
