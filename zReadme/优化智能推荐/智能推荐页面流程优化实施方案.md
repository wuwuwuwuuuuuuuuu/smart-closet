# 智能推荐页面流程优化实施方案

## 1. 问题复述

当前你要做的不是直接重写整套业务，而是**基于现有微信小程序代码，迭代改造“穿搭日记 / 智能提醒”相关页面**，把它落成一个新的“智能推荐”流程。你的目标可拆成 3 个明确产品要求：

1. **智能推荐页面（原穿搭日记页）重做**
   - 最上方：固定主视觉图片
   - 中间：天气提醒，保留现有天气能力，但视觉和文案要重新组织
   - 最下方：人机对话框，用户输入需求，AI 返回推荐结果
   - AI 回复结果卡片中要有一个箭头，点击后跳转到试穿页面

2. **命名统一**
   - 首页中的“穿搭日记”改名为“智能推荐”
   - 页面内“智能提醒”统一改名为“智能推荐”
   - 主视觉图使用：`images/smart_recommend_hero_compressed.jpg`
   - 箭头图使用：`images/67134f606087c7f32d75f9d9f47f5af.png`

3. **试穿页加回跳入口**
   - 从智能推荐页点击 AI 回复卡片里的箭头，要进入试穿页
   - 试穿页右侧增加一个圆形小图标
   - 图标素材使用：`images/smart_recommend_robot_icon.png`
   - 如果该图仍带水印 / 黑边，需要先清理再接入
   - 点击这个圆形图标，可以从试穿页回到智能推荐页

你要求输出的不是直接代码，而是：

- 先复述问题
- 结合当前项目真实情况给出可落地的代码修改方案
- 再拆成适合另一个 agent 执行的若干子任务
- 每个子任务都要写清楚目标、实现方案、异常日志、单测、验证方式
- 文档保存到：`zReadme\优化智能推荐`

## 2. 当前项目实际情况（基于仓库代码）

以下结论是基于当前仓库真实文件，而不是理想状态推测。

## 2.1 当前页面结构

当前路由在 `app.json` 中已经包含：

- `pages/home/home`
- `pages/daily/daily`
- `pages/tryon/tryon`
- `pages/preview/preview`

所以本次优化**不需要新增页面路由**，而应继续沿用现有：

- 首页：`pages/home`
- 智能推荐页（原穿搭日记页）：`pages/daily`
- 试穿页：`pages/tryon`

## 2.2 当前 `pages/daily` 的真实状态

当前 `pages/daily/daily.js` 还是旧版“每日穿搭 / 海报记录页”，核心特征如下：

1. 有日期横向滚动（`dateList / selectDate`）
2. 有海报/图片记录逻辑（`selectedPoster / showUploadOptions / chooseLocalImage / savePosterToAlbum`）
3. 虽然有天气获取逻辑，但页面 `wxml` 里天气内容仍是硬编码：
   - `9/11℃`
   - `小雨`
   - `武汉市`
4. 页面并没有真正的人机对话区，也没有智能推荐结果卡
5. `daily.json` 标题仍然是：

```json
{
  "navigationBarTitleText": "每日穿搭"
}
```

结论：**`pages/daily` 需要做的是“页面职责迁移”**，从“图片记录页”改造成“智能推荐页”。

## 2.3 当前 `pages/home` 的真实状态

## 2.4 当前 `pages/tryon` 的真实状态

## 3. 总体代码修改方案

## 3.1 核心策略

本轮不要推翻项目架构，而是采用**最小入侵式迭代**：

1. `pages/daily` 保留路由不变，但页面职责改成“智能推荐”
2. `pages/home` 只改入口文案，不改路由
3. `pages/tryon` 在现有搭配板基础上增加“从智能推荐进入”和“返回智能推荐”的入口
4. 推荐上下文通过 `wx.setStorageSync` 传递，不通过长 URL 传对象
5. 把易出错的计算逻辑抽成 helper，配套 `tests/`

## 3.2 推荐的数据流

### 智能推荐页 → 试穿页

```text
用户输入需求
→ 智能推荐页构造 payload
→ 调用 AI（或 mock fallback）
→ 得到 recommendationResult
→ 点击结果卡片箭头
→ 写入 wx.setStorageSync('smartRecommendTryonEntry', ...)
→ switchTab / navigate 到 pages/tryon
→ tryon onShow 读取缓存并消费
→ 若有推荐衣物 ID，则自动预选
```

### 试穿页 → 智能推荐页

```text
点击右侧圆形机器人图标
→ navigateTo('/pages/daily/daily')
```

```js
const { logWarning } = require('../../utils/logger')

function normalizeInput(value) {
  return typeof value === 'string' ? value.trim() : ''
}

function normalizeTemperature(tempText) {
  if (typeof tempText === 'number' && Number.isFinite(tempText)) {
    return tempText
  }

  if (!tempText) {
    return null
  }

  const matched = String(tempText).match(/-?\d+(\.\d+)?/)
  if (!matched) {
    logWarning('daily.normalizeTemperature', 'invalid temperature text', { tempText })
    return null
  }

  return Number(matched[0])
}

function buildDateLabel(date = new Date()) {
  const weekMap = ['周日', '周一', '周二', '周三', '周四', '周五', '周六']
  return `${date.getMonth() + 1}月${date.getDate()}日 ${weekMap[date.getDay()]}`
}

function normalizeRecommendationResult(raw = {}) {
  return {
    requestId: raw.requestId || `mock_${Date.now()}`,
    summary: raw.summary || '已根据你的需求生成搭配建议。',
    replyText: raw.replyText || '',
    selectedClothesIds: Array.isArray(raw.selectedClothesIds)
      ? [...new Set(raw.selectedClothesIds.filter(Boolean))]
      : [],
    outfitLines: Array.isArray(raw.outfitLines) ? raw.outfitLines.filter(Boolean) : [],
    tips: Array.isArray(raw.tips) ? raw.tips.filter(Boolean) : [],
    ctaLabel: raw.ctaLabel || '去试穿页继续搭配'
  }
}
```

# 子任务 3：接通天气卡真实数据绑定，去掉所有硬编码天气文案

## 1. 任务目标

保留现有天气获取能力，但页面展示必须改成真实数据绑定，且要生成新的“天气提醒”文案。

## 2. 实现方案

### 2.1 当前真实问题

`daily.js` 已有天气请求，但 `daily.wxml` 仍写死：

- `9/11℃`
- `小雨`
- `武汉市`

### 2.2 建议抽象天气提醒文案

```js
function buildWeatherSuggestion({ temp, text, city }) {
  const temperature = normalizeTemperature(temp)
  const weatherText = text || '未知天气'
  const cityText = city || '当前城市'

  if (temperature === null) {
    return `${cityText}${weatherText}，建议按体感温度灵活搭配。`
  }

  if (weatherText.includes('雨')) {
    return `${cityText}${weatherText}，当前约 ${temperature}°C，建议优先选择防水外套、长裤，并准备雨具。`
  }

  if (temperature <= 10) {
    return `${cityText}${weatherText}，当前约 ${temperature}°C，建议选择保暖内搭、外套和包脚鞋。`
  }

  if (temperature >= 28) {
    return `${cityText}${weatherText}，当前约 ${temperature}°C，建议选择轻薄透气搭配并减少叠穿。`
  }

  return `${cityText}${weatherText}，当前约 ${temperature}°C，适合做通勤或日常层次搭配。`
}
```

```js
onRecommendationInput(e) {
  const value = typeof e.detail.value === 'string' ? e.detail.value : ''
  this.setData({ pendingUserInput: value })
}

appendConversationMessage(message) {
  this.setData({
    conversationList: this.data.conversationList.concat({
      id: `${Date.now()}_${Math.random().toString(16).slice(2)}`,
      ...message
    })
  })
}

async submitRecommendationRequest() {
  const userInput = normalizeInput(this.data.pendingUserInput)
  if (!userInput) {
    wx.showToast({ title: '请先输入需求', icon: 'none' })
    return
  }

  this.appendConversationMessage({ role: 'user', type: 'text', text: userInput })
  this.setData({
    pendingUserInput: '',
    isRecommendationLoading: true
  })

  try {
    const payload = this.buildRecommendationPayload(userInput)
    const result = await this.requestRecommendationWithFallback(payload)
    const normalized = normalizeRecommendationResult(result)

    this.appendConversationMessage({
      role: 'assistant',
      type: 'result-card',
      data: normalized
    })

    this.setData({
      recommendationResult: normalized
    })
  } catch (error) {
    logError('daily.submitRecommendationRequest', error)
    wx.showToast({ title: '生成建议失败', icon: 'none' })
  } finally {
    this.setData({ isRecommendationLoading: false })
  }
}
```
